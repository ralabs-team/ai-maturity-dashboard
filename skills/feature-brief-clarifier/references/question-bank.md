# Feature Question Bank

Use this bank to pick the most relevant 5 to 8 questions for a request. Do not ask all of them. Choose the smallest set that clarifies the feature well.

## Core Questions

1. What user problem or business outcome should this feature solve?
2. Who is the primary user or persona?
3. What should be included in the first version?
4. What is explicitly out of scope?
5. What does success look like from the user's perspective?
6. What acceptance criteria should we use to consider this done?

## UI And Workflow

1. Where in the app should the entry point live?
2. What should the happy-path user flow look like?
3. Are there important empty, loading, or error states?
4. Should this match any existing page or component pattern?
5. Does mobile behavior matter for this feature?

## API And Data

1. Will this require new API routes or changes to existing ones?
2. What data needs to be read, written, or derived?
3. Are there validation rules or business constraints we must enforce?
4. Should this work with existing survey or workspace contracts?
5. Do we need backward compatibility with current payload shapes?

## Permissions And Risk

1. Who should be allowed to see or use this feature?
2. Does it involve sensitive data or workspace-specific restrictions?
3. Are there audit, logging, or analytics requirements?
4. Are there risks if users misunderstand or misuse the feature?

## Delivery Constraints

1. Is there a deadline or release constraint?
2. Do we need to keep this behind a flag or staged rollout?
3. Are there tests or demos that must pass before sign-off?
4. Should we optimize for speed, polish, or minimum viable scope?
